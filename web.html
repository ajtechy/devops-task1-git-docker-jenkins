Welcome to my home page


Automation using ( git + docker + jenkins ) to deploy our webserver



Problem Statement -


1. Create a git repository and develop a Jenkins job that will fetch from master branch on every update to the branch and launch a live httpd container.
2. Create a dev branch and develop a Jenkins job that will fetch from dev branch on every update to the branch and launch a live httpd container for testing environment.
3. Jenkins will test the website running in test environment. If it passes the test, dev branch will be merged to master branch and job 1 will be scheduled.
